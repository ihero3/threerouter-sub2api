# AI Governance & Compliance 升级方案

## 1. 背景与目标

当前系统的 AI Governance & Compliance 模块所有配置均为**全局配置**，所有用户共享同一套规则、模板和 ZDR 策略。

但实际业务模型为：

> 企业注册一个 User Account → 该 Account 创建多个 API Key → 不同员工/系统使用不同 API Key

因此需要将治理配置从**全局**下沉到 **Account 级（User Account）**，使每个企业客户可以拥有独立的合规策略，同时保留平台管理员的全局模板/规则库能力。

## 2. 核心设计原则

```
Platform Admin（平台管理员）
        |
        ↓ 维护全局库
Customer Account（企业主账号） ←→ Compliance Profile
        |
        ├── API Key A（继承 Account 策略）
        ├── API Key B（继承 Account 策略）
        └── API Key C（继承 Account 策略）
```

- **Platform Admin**：维护全局能力开关、行业模板库、内容审核规则库。
- **Customer Account**：当前系统的 `users` 表记录，每个账号即一个企业租户。
- **API Key**：继承 Account 的合规策略，第一阶段不允许单独覆盖。
- **Data Subject**：通过用户端门户行使 GDPR 数据主体权利（同意、导出、删除）。

## 3. 配置归属矩阵

| 配置 | 归属 | 第一阶段 | 备注 |
|------|------|----------|------|
| 行业模板 | Account | ✅ | 用户选择平台预设模板 |
| ZDR 模式 | Account | ✅ | aggregate_only / audit |
| 明细日志保留天数 | Account | ✅ | 配合 ZDR 模式 |
| 合规框架选择 | Account | ✅ | GDPR / EU AI Act / HIPAA 等 |
| 内容审核规则启用/停用 | Account | ✅ | 从平台规则库中选择启用 |
| 内容审核规则自定义 | Account | ❌ | 第二阶段实现 |
| API Key 策略覆盖 | API Key | ❌ | 暂不实现 |
| 用户同意记录 | Data Subject | ✅ 已有 | `user_consents` |
| 数据删除请求 | Data Subject | ✅ 已有 | `data_erasure_requests` |
| 数据导出 | Data Subject | ✅ 已有 | `ComplianceUserDataExporter` |

## 4. 数据模型

### 4.1 新增 `user_compliance_profiles` 表

```sql
CREATE TABLE IF NOT EXISTS user_compliance_profiles (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL UNIQUE REFERENCES users(id) ON DELETE CASCADE,
    active_template_code VARCHAR(30),
    zdr_mode VARCHAR(20) NOT NULL DEFAULT 'aggregate_only', -- aggregate_only | audit
    detail_retention_days INT NOT NULL DEFAULT 7,
    compliance_frameworks JSONB NOT NULL DEFAULT '[]',       -- ["gdpr", "eu_ai_act"]
    moderation_policy JSONB NOT NULL DEFAULT '{"enabled_rule_ids":[]}', -- 启用的规则 ID 列表
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_user_compliance_profiles_user_id ON user_compliance_profiles(user_id);
```

### 4.2 与现有表的关系

| 表 | 关系 | 说明 |
|----|------|------|
| `users` | 1:1 | 一个用户对应一个企业合规档案 |
| `compliance_policy_templates` | N:1 | 平台预设模板，用户引用 `template_code` |
| `moderation_rules` | 多选 | 用户从平台规则库中启用部分规则 |
| `usage_logs` | 生效目标 | 写入时根据 `user_compliance_profiles.zdr_mode` 决定 `aggregate_only` |

### 5.3 用户端 API

前缀：`/api/v1/governance`

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/profile` | 获取当前 Account 的合规档案 |
| PUT | `/profile` | 更新当前 Account 的合规档案 |
| GET | `/templates` | 列出可用行业模板 |
| POST | `/templates/apply` | 应用某个行业模板到当前 Account |
| GET | `/moderation-rules` | 列出平台规则库（供用户选择启用） |
| GET | `/status` | 获取当前 Account 的合规状态 |

### 5.4 ZDR 生效逻辑

写入 `usage_logs` 时：

1. 获取当前 `user_id`
2. 查询 `user_compliance_profiles.zdr_mode` 和 `detail_retention_days`
3. 若 `zdr_mode = 'aggregate_only'`：
   - `aggregate_only = TRUE`
   - `retention_expires_at = NULL`
4. 若 `zdr_mode = 'audit'`：
   - `aggregate_only = FALSE`
   - `retention_expires_at = NOW() + detail_retention_days`

**性能优化**：用户合规档案缓存到内存（TTL 60 秒），避免每次写日志都查库。

### 5.5 内容审核规则过滤逻辑

- 平台规则库仍为全局表 `moderation_rules`
- 每个 Account 在 `moderation_policy.enabled_rule_ids` 中记录启用的规则 ID
- 内容审核时，先加载全局规则，再按当前 `user_id` 过滤只执行启用的规则
- 若 `enabled_rule_ids` 为空或未设置，默认启用所有全局规则（向后兼容）

## 6. 前端改造

### 6.1 用户端 `GovernanceView.vue`

在现有个人数据权利页面基础上，增加 **Account 合规配置** 区域：

1. **行业模板卡片**
   - 列出平台预设模板
   - 显示当前激活模板
   - Apply 按钮

2. **ZDR 设置卡片**

   - 明细保留天数（仅 Audit 模式）

3. **合规框架卡片**

   - EU AI Act
   - HIPAA
   - 可多选

4. **内容审核策略卡片**
   - 列出平台规则库
   - 启用/停用开关

## 7. 管理员端调整

管理员端继续维护全局模板和规则库，但模板表需考虑未来扩展：

```sql
ALTER TABLE compliance_policy_templates ADD COLUMN is_builtin BOOLEAN NOT NULL DEFAULT TRUE;
```

- 平台预设模板：`is_builtin = TRUE`
- 用户自定义模板：`is_builtin = FALSE`（第二阶段）

第一阶段只做平台预设模板，用户只能选择，不能创建。

## 9. 风险与缓解

| 风险 | 说明 | 缓解 |
|------|------|------|
| 性能 | `usage_logs` 写入增加用户配置查询 | 用户合规档案本地缓存（TTL 60s） |
| 向后兼容 | 旧用户无 `user_compliance_profiles` 记录 | 查询不到时返回默认配置（aggregate_only, 7 days） |
| 规则引擎 | 全局规则按用户过滤增加计算量 | 规则数量可控，过滤逻辑轻量 |
| 权限边界 | 用户不能修改全局规则，只能选启用哪些 | Service 层严格校验 |

## 10. 默认值

新用户注册或首次访问治理配置时，自动创建默认档案：

```json
{
  "active_template_code": null,
  "zdr_mode": "aggregate_only",
  "detail_retention_days": 7,
  "compliance_frameworks": ["gdpr"],
  "moderation_policy": {
    "enabled_rule_ids": []
  }
}
```

即：**默认启用 ZDR、选择 GDPR、启用所有全局规则**。
